# San Felicissimo Dashboard

Operational dashboard system designed to collect, process, and visualize data from multiple sources through automated pipelines, a continuously updated database, and custom web/iOS interfaces.

## What it does

- Collects data from multiple operational sources such as management platforms and analytics tools
- Processes and normalizes incoming data through automated scripts
- Stores updated information inside a continuously refreshed database
- Displays key metrics through custom web and iOS interfaces

## System structure

- **Data ingestion layer**  
  Automated scripts gather data from multiple external and operational sources.

- **Processing layer**  
  Incoming data is cleaned, normalized, and prepared for structured use.

- **Storage layer**  
  Processed information is stored in a continuously updated database.

- **Presentation layer**  
  Metrics and operational indicators are exposed through custom web and iOS interfaces.

## Why it matters

This system turns fragmented operational data into a structured, continuously updated dashboard environment.  
Its purpose is not just to display numbers, but to create a usable decision layer across multiple sources, interfaces, and workflows.

## Status

Active internal project with working data pipelines, database updates, and custom interface layers.  
Further refinements may continue as the operational system evolves.

## Scope

This project is designed as a focused operational system for a specific real-world environment.  
It is not intended as a generic SaaS product, but as a custom internal architecture built around concrete workflows, data flows, and decision support needs.
